Well done! We will now add visual feedback for our user to better understand what is happening.

Our feedback strategy will combine two ways

feedback on gates
we will expect the gates to toggle their css class between free and occupied
a #info section
we will expect an info element to toggle its css class between hidden and visible

So, starting from what we have so far

  • #gate-1 should start with css class containing free
  • #gate-1 should have its css class contain occupied when a first ship is docked
  • #info should start with its css class containing hidden
  • #info should have its css class not to contain hidden when a first ship is docked
  • #info should have its css class reset to hidden when one starts to type a new ship name in the input#ship element
  • No verification will be made on the content of your #info element, to let you choose the content that makes sense for you
